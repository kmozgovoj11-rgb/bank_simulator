# AGENTS.md

## ОБЯЗАТЕЛЬНО

- Считать `Customer` и `User` разными сущностями. Не объединять их.
- Считать телефон логином пользователя. Сохранять текущую нормализацию и валидацию телефона.
- Сохранять текущие правила валидации пароля.
- Сохранять хранение пароля в виде хеша. Не хранить пароль в открытом виде.
- Сохранять атомарность перевода через `TransactionBroker`.
- Сохранять rollback-логику транзакций.
- Сохранять ограничения по счетам:
  - нельзя выполнять операции по счетам в статусах `FROZEN` и `CLOSED`;
  - нельзя закрывать счет с ненулевым балансом;
  - нельзя переводить деньги на тот же самый счет.
- Держать SQLite-специфику только в infrastructure persistence.
- Предпочитать маленькие точечные изменения вместо больших рефакторингов.
- Переиспользовать существующие сервисы и валидацию, а не дублировать логику.
- Держать бизнес-правила в domain/application, а не в коде запуска.
- Если меняется persistence-логика, не ломать существующие контракты репозиториев.

## СНАЧАЛА СОГЛАСОВАТЬ

- Изменение схемы базы данных.
- Добавление build tool для сборки и тестирования проекта.
- Добавление новых зависимостей.
- Переименование пакетов.
- Крупный рефакторинг через несколько слоев.
- Изменение формата логина или правил аутентификации.

## НЕЛЬЗЯ

- Добавлять frontend-структуру или frontend-файлы без явного запроса.
- Коммитить локальные файлы базы данных.
- Заменять рабочие бизнес-правила временными заглушками.
- Переносить SQL в domain или application.
