# BankSimulator

**Симуляор банка** - учебный проект, который демонстрирует принципы ООП через реализацию полноценного симулятора банка с тремя типами счетов, системой транзакций, начислением процентов и сохранением данных после закрытия программы.

## Основные функции

- **Управление счетами:** Поддержка трёх типов банковских счетов (дебетовый, депозитный, кредитный) с уникальной логикой начисления средств и комиссий.
- **Транзакции и переводы:** Возможность перевода средств между счетами по уникальному номеру счета, генерация истории операций.
- **Аутентификация и роли:** Вход в систему по логину и паролю с разграничением прав (клиент, администратор, оператор) для демонстрации безопасности.
- **Персистентность данных:** Сохранение состояния счетов и пользователей в базе данных (БД) после закрытия программы, что позволяет восстанавливать сессию при повторном входе.
- **Надёжность и обработка ошибок:** Валидация входных данных, проверка достаточности средств, журналирование действий пользователя и защита от некорректных операций.
- **Расширяемость:** Архитектура позволяет легко добавлять новые типы счетов, комиссий или банковских продуктов без изменения ядра системы.

## Технический стек

- **Язык программирования:** Python / Java — строгая типизация и поддержка ООП.
- **Инструменты для диаграмм:** PlantUML (для построения UML-диаграммы классов и отображения архитектуры проекта).
- **Архитектура:** Многослойная с разделением на модели, сервисы и репозитории.
- **Принципы и паттерны:** SOLID, применение порождающих и структурных паттернов (Factory, Singleton, Observer) для гибкого создания объектов и обработки событий.
- **Хранение данных:** Реляционная БД (PostgreSQL) для сохранения пользователей, счетов и истории операций.
- **Тестирование:** Модульные тесты для проверки core-логики, рефакторинг и код-ревью на всех этапах разработки.
